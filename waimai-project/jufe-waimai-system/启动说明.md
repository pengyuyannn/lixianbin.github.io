# 校园外卖系统 - 启动说明

## 环境要求
- JDK 17+
- MySQL 8.0+
- Redis 6.0+
- Maven 3.6+

## 数据库配置

### 1. 创建数据库
```sql
CREATE DATABASE IF NOT EXISTS jufe DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 导入数据表结构
使用提供的 `jufe@localhost--jufe.sql` 文件创建所有数据表。

### 3. 修改数据库连接配置
编辑 `src/main/resources/application.yml`，修改数据库连接信息：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/jufe
    username: root
    password: 你的数据库密码
```

## Redis 配置

### 1. 安装 Redis
- macOS: `brew install redis`
- Ubuntu: `sudo apt-get install redis-server`
- Windows: 下载 Redis for Windows 或使用 WSL

### 2. 启动 Redis
```bash
redis-server
```

### 3. 验证 Redis 连接
```bash
redis-cli ping
# 应返回 PONG
```

## 启动项目

### 1. 清理和编译
```bash
mvn clean compile
```

### 2. 打包（可选）
```bash
mvn clean package -DskipTests
```

### 3. 运行应用
方式一：使用 Maven
```bash
mvn spring-boot:run
```

方式二：运行主类
在 IDE 中运行 `JufeWaimaiSystemApplication.java`

方式三：使用 jar 包
```bash
java -jar target/jufe-waimai-system-0.0.1-SNAPSHOT.jar
```

## 验证启动成功

启动成功后，访问以下接口测试：

```
GET http://localhost:8080/api/merchants

### 2. 商品列表接口
```

### 3. 购物车接口

## API 接口文档

### 商家管理
- `GET /api/merchants` - 获取商家列表
- `GET /api/merchants/{id}` - 获取商家详情
- `POST /api/merchants` - 创建商家
- `PUT /api/merchants` - 更新商家

### 商品管理
- `GET /api/products/merchant/{merchantId}` - 获取商家商品列表
- `GET /api/products/{id}` - 获取商品详情
- `POST /api/products` - 创建商品
- `PUT /api/products` - 更新商品
- `PUT /api/products/{id}/stock` - 更新商品库存

### 购物车管理
- `GET /api/cart/student/{studentId}` - 获取购物车列表
- `POST /api/cart` - 添加到购物车
- `PUT /api/cart/{id}` - 更新购物车商品数量
- `PUT /api/cart/{id}/toggle` - 切换选中状态
- `DELETE /api/cart/student/{studentId}` - 清空购物车

### 订单管理
- `POST /api/orders` - 创建订单
- `POST /api/orders/{id}/pay` - 支付订单
- `POST /api/orders/{id}/merchant-accept` - 商家接单
- `POST /api/orders/{id}/rider-accept` - 骑手接单
- `POST /api/orders/{id}/start-delivery` - 开始配送
- `POST /api/orders/{id}/complete` - 完成订单
- `POST /api/orders/{id}/cancel` - 取消订单
- `GET /api/orders/{id}` - 获取订单详情
- `GET /api/orders` - 获取订单列表

## Redis 缓存优化说明

本系统在以下地方使用了 Redis 缓存优化：

### 1. 商家信息缓存
- Key: `merchant:{id}`
- 过期时间：1 小时
- 场景：查询商家详情时先查缓存，缓存不命中再查数据库

### 2. 商品信息缓存
- Key: `product:{id}` - 单个商品信息
- Key: `products:merchant:{merchantId}` - 商家商品列表
- 过期时间：30 分钟
- 场景：查询商品详情和商品列表时使用缓存

### 3. 购物车缓存
- Key: `cart:student:{studentId}`
- 过期时间：30 分钟
- 场景：获取购物车列表时使用缓存

### 4. 订单信息缓存
- Key: `order:{id}` - 单个订单信息
- Key: `orders:student:{studentId}` - 学生订单列表
- 过期时间：2 小时
- 场景：查询订单详情和订单列表时使用缓存

### 缓存一致性保证
- 当数据更新时，会删除对应的缓存，确保下次查询时从数据库加载最新数据
- 使用事务保证订单创建、支付等操作的数据一致性

## 常见问题

### 1. Redis 连接失败
检查 Redis 是否启动，端口是否正确（默认 6379）

### 2. 数据库连接失败
检查 MySQL 服务是否启动，数据库名、用户名、密码是否正确

### 3. 端口冲突
如果 8080 端口被占用，可以在 `application.yml` 中修改：
```yaml
server:
  port: 8081
```
