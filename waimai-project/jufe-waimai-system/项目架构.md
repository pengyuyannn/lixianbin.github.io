# 校园外卖系统 - 项目架构文档

## 项目概述
基于 Spring Boot + MyBatis + Redis 的校园外卖系统，支持学生、商家、骑手三种角色，实现完整的点餐、配送流程。

## 技术栈
- **后端框架**: Spring Boot 3.5.11
- **持久层**: MyBatis 3.0.5
- **缓存**: Redis (Lettuce 连接池)
- **数据库**: MySQL 8.0+
- **工具类库**: Lombok
- **构建工具**: Maven

## Redis 缓存策略

### 缓存 Key 设计规范
```
merchant:{id}                    # 商家信息
product:{id}                     # 商品信息
products:merchant:{merchantId}   # 商家商品列表
cart:student:{studentId}         # 学生购物车
order:{id}                       # 订单信息
orders:student:{studentId}       # 学生订单列表
```

### 缓存过期时间
- 商家信息：1 小时
- 商品信息：30 分钟
- 购物车：30 分钟
- 订单信息：2 小时

### 缓存一致性保证
1. **查询操作**: 先查缓存，缓存不命中则查数据库并回写缓存
2. **更新操作**: 更新数据库后删除对应缓存
3. **删除操作**: 直接删除缓存

### 适用场景分析
1. **商家信息**: 读多写少，适合缓存
2. **商品信息**: 读多写少，适合缓存
3. **购物车**: 频繁读写，使用缓存减少数据库压力
4. **订单信息**: 查询频繁，适合缓存

## 订单状态流转

```
0-待支付 → 1-已支付待接单 → 2-商家已接单 → 3-骑手已接单 → 4-配送中 → 5-已完成
         ↓
        6-已取消
```

### 状态说明
- **0-待支付**: 订单创建，等待支付
- **1-已支付待接单**: 支付成功，等待商家接单
- **2-商家已接单**: 商家已确认订单
- **3-骑手已接单**: 骑手已接单准备配送
- **4-配送中**: 骑手正在配送
- **5-已完成**: 订单完成
- **6-已取消**: 订单取消
- **7-退款中**: 申请退款

## API 接口设计

### RESTful 风格
- GET - 查询操作
- POST - 创建操作
- PUT - 更新操作
- DELETE - 删除操作

### 接口规范
- 所有接口统一返回 `Result<T>`格式
- 成功返回 code=200, message="success"
- 失败返回 code=500, message=错误信息

## 事务管理

### 使用@Transactional 的场景
1. 创建订单（涉及订单主表、订单详情表、购物车表）
2. 支付订单（更新订单状态和时间）
3. 订单状态变更（接单、配送、完成）
4. 取消订单（更新状态、恢复库存）

### 事务回滚
- 使用`@Transactional(rollbackFor = Exception.class)`确保所有异常都回滚

## 安全设计

### 密码加密
- 使用 BCrypt 加密算法
- 密码不明文存储

### SQL 注入防护
- 使用 MyBatis 参数化查询
- 避免字符串拼接 SQL

### XSS 防护
- Controller 层参数校验
- 输出转义

## 性能优化

### 数据库优化
1. 索引优化（phone, username, order_no 等关键字段）
2. 外键约束保证数据一致性
3. 适当分表（订单详情独立表）

### Redis 优化
1. 连接池配置（max-active: 8, max-idle: 8）
2. JSON 序列化提高可读性
3. 合理设置过期时间

### 代码优化
1. Service 层缓存逻辑封装
2. 批量操作（订单详情批量插入）
3. 快照机制（订单详情保存商品信息快照）
